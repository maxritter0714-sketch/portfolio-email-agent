# portfolioAgent — CLAUDE.md

## Project
**Portfolio Email Agent** — sends an HTML portfolio report every second Saturday.
Stack: Python · yfinance · NewsAPI · Anthropic Claude · Gmail SMTP

## Key files
| File | Purpose |
|---|---|
| `agent.py` | Main agent: fetches prices, news, runs Claude analysis, sends email |
| `import_finanzfluss.py` | Converts Finanzfluss export CSV → `portfolio.csv` |
| `finanzfluss_export.csv` | Latest Finanzfluss export (update manually, then re-run import) |
| `portfolio.csv` | Active holdings in yfinance format (generated by import script) |
| `.env` | API keys and Gmail credentials (never commit) |
| `requirements.txt` | Python dependencies |

## Workflow — updating holdings
1. Export from Finanzfluss → save as `finanzfluss_export.csv`
2. Add any new ISINs to `ISIN_TO_TICKER` in `import_finanzfluss.py`
3. Run: `python import_finanzfluss.py --input finanzfluss_export.csv`
4. Verify `portfolio.csv` looks correct
5. Test: `python agent.py --force`

## ISIN map rules
- Hardcoded map in `import_finanzfluss.py` is the primary resolution path
- Fallback: yfinance Search API (slow, unreliable — prefer adding to map)
- Zertifikate/OS positions are always excluded
- Positions with `Anzahl = 0` are skipped (sold/zero positions kept for history)

## Commit style
Conventional Commits: `type(scope): description`
Examples: `feat(import): add Fluence Energy ISIN`, `fix(agent): handle missing ticker`

## Session logs
Stored in `.claude/sessions/YYYY-MM-DD-slug.md`
Run `/session-start` at the beginning of each session.
Run `/session-end` at the end to write the handoff log.
